# 開發計畫

## 目標

建立一個純前端網頁工具，比較兩份 Palo Alto 防火牆 XML 設定檔，提供 36 項稽核結果與 Excel 報表，且設定內容不得離開使用者電腦。

## 已完成

- 雙 XML 上傳與瀏覽器端 XML 解析。
- A/B Hostname、版本與 vSys／Device Group 偵測及選擇。
- 36 項比對導覽、完整結果表、搜尋、狀態篩選與「僅看差異」。
- A/B 對等比較：一致、變更、僅存在 A、僅存在 B、順序提示。
- Excel 匯出：摘要與 36 個明細分頁。
- 項目 1 至 6 的擷取路徑修正：HA、Permitted IP、系統／Logging、Network Profiles、Interface 與靜態路由。
- 操作手冊與專案維護文件。

## 下一階段優先順序

1. 使用四組 `CONFIG` 範例逐項驗證 36 個項目，記錄實際 PA XML 路徑與預期筆數。
2. 將項目 1、2、3 改為使用者可讀的固定欄位名稱，而非只顯示 XML dotted path。
3. 補強 Panorama Device Group 的 `pre-rulebase`／`post-rulebase` 政策擷取與標示。
4. 驗證 shared 與 vSys 同名物件的覆寫規則與報表呈現方式。
5. 強化 Excel 樣式：差異列底色、凍結首列、欄位篩選與摘要超連結。
6. 針對大量 Address／Policy 導入分頁或虛擬捲動，並做效能測試。

## 驗收準則

- 上傳兩份有效 PA XML 後可在合理時間內完成比對。
- 每項結果都能顯示 A 值、B 值、狀態與繁體中文說明。
- 「一致」項目仍保留，符合完整稽核需求。
- 敏感欄位不顯示明碼，但能反映是否不同。
- 匯出的 `.xlsx` 可正常開啟，含摘要與各項明細。

## 開發原則

- 優先修正擷取正確性，再處理視覺美化。
- 不以 A/B 推論新舊版本；雙機設定視為對等。
- 每次新增 XML 路徑時，應同時在範例檔驗證。
- 不新增任何會將 XML 上傳、持久化或傳送至第三方的功能。
